Self-Introduce

1년 6개월 동안 몸 담고 있었던 곳들에서 수행한 업무에 대한 대략적인 진술입니다.

주요 업무

주업무는 HTML 과 CSS 를 사용하여 웹사이트를 제작했으며 java script 와 jquery 를 사용해서 동적인 작업을 처리했습니다. 특히 자바스크립트보다는 제이쿼리를 통해 동적인 작업을 처리했으며, 슬라이드는 주로 swiper 라이브러리를 사용했습니다. 아무래도 자바스크립트보다 제이쿼리가 훨씬 익숙합니다. 제이쿼리 또한 자바스크립트의 기능을 좀 더 빠르고 쉽게 구현하기 위한 목적을 갖고 있는 만큼 편하게 작업을 했었습니다. 그런데 요새는 제이쿼리와 같은 라이브러리를 최대한 지양하는 즉슨, 바닐라 자바스크립트를 선호하는 추세라고 알고 있습니다. 그래서 추세에 맞게 자바스크립트를 사용해서 기능을 구현하는 방식으로 작업할 생각입니다.

사용 툴

나인원랩스에 있을 때는 EDITPLUS 라는 툴을 활용했습니다. 에디트플러스의 장점은 FTP 가 편리하다는 것이었는데 개인적으로는 VISUAL STUDIO CODE 가 좀 더 편하고 또한 시각적으로도 보기 좋다고 생각했습니다.

반응형

일을 시작하고 2달 가량은 적응형으로 작업을 처리했었는데, 개발 코드를 입히거나 수정 작업을 진행할 때 같은 작업을 두 번 해야한다는 번거로움 때문에 반응형 작업 방식으로 변경했습니다. 반응형 작업은 디자인이 어긋나는 지점마다 미디어 쿼리 값을 줘서 처리를 했었습니다. 그러다가 일을 하고 대략 10개월이 지나고 나서 어차피 작업을 좀 더 빠르고 효율적으로 처리하기 위한 방법에 대해 고민했었습니다. 그리고 결론은 viewport 를 이용해서 작업을 처리하는 것이었습니다. 실제로 삼성전자 홈페이지가 이와 같은 방식으로 작업이 되어 있으며, 이를 참조해서 컨텐츠 영역에 한 번 값을 주고 768px (태블릿 사이즈)을 기준으로 값을 주어서 모바일 반응형 작업을 진행했습니다.

크로스 브라우징

나인원랩스에서는 9개월 가량은 레이아웃을 정렬할 때 float 나 inline-block를 사용했습니다. css flex 나 grid 는 작년 2월 즘에 처음 사용하게 되었었는데, 그 이유는 인터넷 익스플로러 6에서는 flex 나 grid 를 사용할 수 없었기 때문입니다. 그런데 아시다시피 flex 가 편의성으로 따지면 혀를 내두를 정도로 아주 편하다는 것을 알게 되었습니다. 그래서 당시에 컨택업체의 특성 및 사용자의 브라우저 사용 비율에 따라 정렬 방식을 다르게하는 것이 좋을 것이라 건의를 드려서 작업 방식을 변경했었습니다.

물론 지금은 너무나 다행히도 익스플로러 서비스가 종료 되어서 이런 문제를 겪지 않고 편하게 flex 코드로 작업을 하고 있습니다. 그리고 메타올로 이직 후에는 아무래도 NFT 관련 업체다 보니 주로 크롬 브라우저를 사용했어서 크로스 브라우징을 크게 신경쓰지 않았으며, 간혹 IOS 기기 관련해서 크로스 브라우징 작업을 진행했었습니다.

REACT

근래에 시간이 날 때마다 리액트 공부를 하고 있습니다. 무얼 하던 간에 지금 당장 필요한 일을 하는 것이 중요하며 또한 목적을 갖고 일에 임하는 것이 중요하다고 생각합니다. 종종 머리속에 있는 내용을 정리하는 습관을 갖고 있는데, 아무래도 정리한 내용들을 효율적으로 관리하려면 저에게 필요한 기능들을 종합한 프로그램이 있으면 좋겠닥 생각했습니다. 그리고 이왕이면 역량을 키울 수 있는 일이면 좋겠다 생각해서 작년 11월 즈음부터 틈틈히 공부를 하고 있습니다. 그리고 이를 토대로 서버 관련 공부도 할 생각입니다.

Front-end

기존에는 HTML, CSS, Java script 그리고 JQuery 를 활용해서 퍼블리싱 작업만 해 왔습니다. 그런데 아무래도 계속 이 일을 하기엔 지루하기도 하고 앞서 언급한 역량의 개발에 집중해야겠다 생각해서 프론트 쪽 공부를 하는 중입니다. API 문서에 있는 정보를 토대로 서버와 통신하는 방법에 대한 아주 기초적인 공부를 하는 중입니다. 실전에서 얼마나 잘 해낼 수 있을지 의문이지만 시간과 정성을 들이면 어렵지 않게 해낼 수 있다고 생각합니다.
